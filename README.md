# Laboratorio de Git y GitHub

Este repositorio hace parte de la capacitación práctica de **Git y GitHub**.

El objetivo de estos laboratorios es que los participantes practiquen el flujo de trabajo colaborativo utilizando Git y GitHub, incluyendo:

* Control de versiones con Git
* Trabajo con ramas
* Pull Requests
* Revisión de código
* Gestión de tareas con Issues
* Resolución de conflictos

## Reglas del laboratorio

1. No hacer **push directo a la rama `main`**.
2. Todo cambio debe realizarse mediante **Pull Request**.
3. Cada Pull Request debe tener **al menos un revisor**.
4. No aprobar tu propio Pull Request.

## Estructura del repositorio

participantes/
Carpetas con la información de cada participante.

## Flujo de trabajo

1. Crear una rama
2. Realizar cambios
3. Hacer commit
4. Hacer push
5. Crear Pull Request
6. Revisar cambios
7. Hacer merge
